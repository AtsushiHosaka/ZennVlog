# ProjectList Feature

## 概要
全プロジェクトのリスト表示画面。タブバーの2番目のタブ。

## 状態（ViewModel）

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| projects | [Project] | 全プロジェクト一覧 |
| isLoading | Bool | ローディング状態 |
| errorMessage | String? | エラーメッセージ |

## ユーザーアクション

1. **プロジェクトカードをタップ** → 撮影画面へ遷移

## 遷移先

- RecordingView（NavigationLink）

## UseCase

- `FetchProjectsUseCase` - 全プロジェクトを取得（更新日順ソート）

## コンポーネント

- `ProjectCard` - プロジェクトカード（サムネイル、名前、テーマ、ステータス、更新日）
  - **レイアウト**: 横型カード（左：サムネイル16:9角丸、右：情報）
  - **表示情報**: プロジェクト名（太字）、テーマ（グレー）、ステータスバッジ（進行中：青、完成：緑）、更新日時（相対表示：「2時間前」「昨日」など）
  - **タップ動作**: RecordingViewへ遷移

## UI/UX詳細

### リストスタイル
- 縦スクロールリスト、セパレータ付き
- プルtoリフレッシュ対応

### 空状態
- 「プロジェクトがありません」メッセージ
- 「新しいプロジェクトを作成する」ボタン → ChatView遷移

## データフロー

画面表示 → ProjectListViewModel.onAppear() → FetchProjectsUseCase → ProjectRepository.fetchAll() → updatedAt降順でソート → @Published変数更新 → SwiftUI描画

## 状態管理とロジック

### ソート
デフォルト: `updatedAt`降順（最新が上）
将来的に追加可能: 名前順、作成日順

### リフレッシュ
プルtoリフレッシュでFetchProjectsUseCaseを再実行

### 検索/フィルタリング（将来的な拡張）
- ステータスでフィルタ（進行中のみ、完成のみ）
- テキスト検索（プロジェクト名、テーマ）

## エラーハンドリング

- **取得失敗**: エラーメッセージ表示、「再試行」ボタン
- **空状態**: 「プロジェクトがありません」+ 新規作成ボタン

## 注意点

- プロジェクト詳細画面は設けず、直接撮影画面に遷移
- リストは更新日順でソート
- プルtoリフレッシュで最新のプロジェクト一覧を取得
